这份快速开始面向第一次接触 AOS Desktop 的团队成员，目标是在几分钟内完成第一次扫描并理解结果页面。

## 第一步：安装并启动客户端

安装对应平台的桌面客户端后启动应用，进入主工作台。AOS Desktop 基于 Tauri 构建，开发与打包链路覆盖 macOS、Windows 和 Linux 桌面环境。

## 第二步：完成第一次本地扫描

桌面端会先做本地发现，再执行首轮静态规则检测。当前扫描链路由两部分组成：

- `discovery-engine`：生成统一的本地组件清单
- `local-scanner`：对内容做首轮静态风险检测

默认重点关注的本地路径包括：

- `~/.openclaw/skills`
- `~/.agents/skills`
- `~/.cline/plugins`

当前支持重点识别的文件类型包括：

- `.py`
- `.js`
- `.ts`
- `.sh`
- `.md`

## 第三步：阅读结果

第一次扫描完成后，优先关注三类信息：

- 总体风险分与 verdict：帮助你先判断当前环境是否需要立刻处置。
- 高风险组件列表：优先进入风险分析页查看具体 findings。
- 风险类型分布：看问题集中在命令执行、文件访问、外联请求还是提示注入。

## 第四步：对高风险组件做处置

对可疑 Skill，建议按下面的顺序处理：

1. 打开风险详情页，确认具体文件、规则命中和建议动作。
2. 如果短期内不打算继续使用，先移入隔离区。
3. 如果仍需保留，结合团队流程做代码复核或二次验证。

## 第五步：补充两类进阶检查

- 代码库扫描：适合检查本地 Agent 项目、MCP 配置和脚本执行链路。
- 活动监控：适合在真实运行过程中观察工具调用、模型调用和安全事件。
